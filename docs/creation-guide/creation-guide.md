# 卡片创作指南

> CardPack Studio 核心创作方法论
> 让知识创作更系统、更高效、更有价值

## 🎯 创作理念

### 核心原则
1. **问题导向**：以问题为核心，引发深度思考
2. **结构简洁**：双层结构，信息密度适中
3. **视角多元**：三个维度，全面覆盖
4. **可操作性**：引导行动，而非空谈理论

### 价值主张
- **个人成长**：系统化整理知识，提升思考质量
- **知识分享**：结构化内容，便于传播和学习
- **决策支持**：提供决策框架，降低试错成本
- **终身学习**：持续迭代，适应时代变化

## 📋 卡片结构规范

### 双层结构标准

```
🎴 001

[问题层：独立完整的核心问题，约35字]

[扩展层：3个延伸小问连续不分行，约100字]
```

#### 问题层规范

**要求**：
- **字数**：35字左右（20-50字）
- **类型**：独立完整的核心问题
- **作用**：引发思考的钩子
- **风格**：直接、具体、有针对性

**示例**：
- ✅ "在AI时代，创业成功的关键到底变成了什么？"
- ✅ "你是在「解决问题」还是「创造问题」？"
- ✅ "当你的产品让用户变得更焦虑，这是好事还是坏事？"

**避免**：
- ❌ 过于宽泛："什么是创业？"
- ❌ 过于简单："创业难吗？"
- ❌ 过于技术化："如何使用AI技术创业？"

#### 扩展层规范

**要求**：
- **字数**：100字左右（80-150字）
- **构成**：3个连续小问，不分行
- **角度**：个人经验、理论原则、批判反思
- **风格**：连贯、深入、多角度

**结构**：
1. **第一问**：个人经验角度（"你最近..."）
2. **第二问**：理论原则角度（"如果..."）
3. **第三问**：批判反思角度（"为什么..."）

**示例**：
```
你个人觉得，自己现在最需要提升的能力是什么？是写代码，还是做判断？如果未来创业者的成功不再依赖于技术深度，那现在大量学习技术还有意义吗？当「能做出来」不再是优势时，什么才可能成为新的决定性壁垒？
```

**避免**：
- ❌ 使用列表符号（- • 1. 2. 3.）
- ❌ 分行显示
- ❌ 重复相同角度的问题
- ❌ 过于抽象或理论化

## 🎨 创作方法论

### 1. 问题发现技巧

#### 从痛点出发
- **观察生活**：记录日常遇到的决策困难
- **反思经验**：从过往成功/失败中提炼问题
- **关注趋势**：结合时代变化提出新问题

#### 从理论出发
- **经典理论**：将传统理论转化为现代问题
- **跨界思考**：将其他领域的方法应用到当前领域
- **逆向思维**：从相反角度思考问题

#### 从实践出发
- **用户反馈**：从用户需求中发现问题
- **数据驱动**：通过数据分析发现问题
- **实验验证**：通过实验验证问题假设

### 2. 问题设计技巧

#### 问题类型
1. **开放性问题**：引发多角度思考
2. **对比性问题**：突出关键差异
3. **假设性问题**：探索可能性
4. **反思性问题**：深度自我审视

#### 问题层次
- **表面问题**：直接可见的现象
- **深层问题**：背后的本质原因
- **系统问题**：整体结构的影响
- **元问题**：关于思考本身的问题

#### 问题表达
- **简洁明了**：避免冗长和复杂
- **具体可感**：避免抽象和模糊
- **引发思考**：避免简单的是非题
- **时代适配**：符合当前时代背景

### 3. 扩展层设计技巧

#### 三个维度设计
1. **个人经验维度**：
   - "你最近..."
   - "你曾经..."
   - "你如何..."

2. **理论原则维度**：
   - "如果..."
   - "根据..."
   - "从...角度看..."

3. **批判反思维度**：
   - "为什么..."
   - "是否..."
   - "真的...吗？"

#### 问题串联技巧
- **逻辑递进**：从具体到抽象
- **视角转换**：从个人到集体
- **时间维度**：从过去到未来
- **深度递进**：从现象到本质

#### 语言表达技巧
- **自然流畅**：避免生硬转折
- **口语化**：避免过于书面化
- **简洁有力**：避免冗余表达
- **引发共鸣**：避免过于专业或学术

## 📊 创作流程

### 1. 选题阶段
```
确定领域 → 收集资料 → 发现问题 → 评估价值
```

### 2. 设计阶段
```
设计问题层 → 设计扩展层 → 验证结构 → 优化表达
```

### 3. 审核阶段
```
内容审核 → 格式检查 → 质量评估 → 用户测试
```

### 4. 发布阶段
```
格式转换 → 质量保证 → 分发发布 → 收集反馈
```
