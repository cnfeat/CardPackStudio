欢迎为 CardPack Studio 贡献您的智慧和创意！本文档将指导您如何参与项目贡献。

## 🎯 贡献方式

### 1. 卡包内容贡献

#### 创建新卡包
1. **选择领域**：选择您擅长的领域（创业、学习、生产力等）
2. **遵循规范**：遵循卡片创作规范和格式要求
3. **质量保证**：确保内容专业、准确、有价值
4. **提交审核**：通过 Pull Request 提交审核

#### 改进现有卡包
1. **发现问题**：识别现有卡包的问题或改进点
2. **提出建议**：在 Issues 中提出改进建议
3. **直接改进**：直接提交改进的 Pull Request

## 📋 贡献规范

### 代码规范

#### Git 提交规范
```
类型(范围): 描述

详细说明（可选）
```

**类型说明：**
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建或辅助工具变动

#### 代码风格
- 使用 2 空格缩进
- 遵循 JavaScript Standard Style
- 添加必要的注释
- 编写测试用例

### 内容规范

#### 卡片创作规范
1. **问题层**：35字左右，独立完整的核心问题
2. **扩展层**：100字左右，3个连续小问
3. **视角多元**：个人经验、理论原则、批判反思
4. **语言简洁**：避免冗长和复杂表达

#### 质量标准
- **专业性**：内容准确，符合专业标准
- **实用性**：能够指导实际思考和行动
- **时效性**：内容不过时，具有长期价值
- **原创性**：尊重知识产权，避免抄袭

### 文档规范

#### Markdown 格式
- 使用标准 Markdown 语法
- 添加适当的标题层级
- 使用代码块标注代码
- 添加链接和引用

#### 文档结构
- 清晰的目录结构
- 统一的文档格式
- 必要的示例和说明
- 相关的参考资料

## 🚀 贡献流程

### 1. Fork 仓库
```bash
# Fork 到您的 GitHub 账户
# 克隆到本地
git clone https://github.com/您的用户名/CardPackStudio.git
cd CardPackStudio
```

### 2. 创建分支
```bash
# 创建功能分支
git checkout -b feature/新功能名称
```

### 3. 开发和测试
```bash
# 开发代码
# 运行测试
npm test
```

### 4. 提交更改
```bash
# 添加更改
git add .

# 提交更改
git commit -m "feat: 添加新功能"
```

### 5. 推送到分支
```bash
git push origin feature/新功能名称
```

### 6. 创建 Pull Request
- 在 GitHub 上创建 Pull Request
- 填写详细的更改说明
- 关联相关的 Issues
- 等待审核和反馈
